1. Global Authentication & Role Routing

The system utilizes State-Driven Routing. The entire app experience is gated by the AuthProvider.

Step 1: User submits credentials via Login.jsx.

Step 2: Backend validates and returns a **JWT Access Token** + User Profile Data.

Step 3: AuthContext stores the token in LocalStorage and sets the currentUser state.

Step 4: ProtectedRoute.jsx evaluates profile.role and allows access to /admin, /trainer, or /user routes.

2. Admin Dashboard Modules

A. Member Central (Members.jsx)

The centralized hub for all gym attendees.

ActionTypeLogic Description
Search/FilterUIInstant filtering using useMemo across memory-cached roster.
Plan ReminderAPIDispatches POST to remind_plan/; triggers Signal-to-Email loop.
Lifecycle MgmtAPIEdit plans, delete accounts, or update trainer assignments.

B. Trainer Directorate (Trainer.jsx)

Controls the personnel staffing the facility.

Assignment Workflow: Admin creates a trainer profile -> Selects "Active" status -> Assigns to Member. The member receives an automated notification: "A new trainer has been assigned to your profile!"

C. Broadcast Command (Broadcast.jsx)

The heavy-duty marketing engine.

Audience Logic: The Admin select-input allows choosing "Gold Only", "Diamond Only", or "All Members". The backend retrieves emails strictly for members whose plan__name matches the selection.

3. Trainer Dashboard Modules

A. Client Roster

Unlike Admin tools, Trainer views are Strictly Isolated. They can only see members where member.trainer.id == currentTrainer.id.

B. Fitness Planning (Workouts & Diets)

Workflow: Trainer selects a member -> Inputs Exercises/Meal Plans -> Saves to WorkoutPlan model. Once saved, the Member Dashboard instantly populates with the latest data.

C. Attendance Scribe (Attendance.jsx)

Trainers log daily check-ins. Each log entry is timestamped and categorized as "Present", "Absent", or "Excused", feeding directly into the analytical charts.

4. Member Dashboard Experience

Members have a read-focused dashboard prioritized for consumption and motivation.

Component Roles:

  • Progress Chart: Visualizes weight/performance metrics over time.
  • My Plan: Displays active subscription tier and countdown to expiry.
  • My Routine: Fetches Workout/Diet plans assigned by their specific trainer.

5. The Communication Engine (Signals)

This backend automation is the "glue" of the project. It uses Django post_save Signals.

Trigger: Any instance created in the Notification table.

Process: Background worker captures the user.email, the title, and message.

Result: An HTML email is constructed and dispatched using email_utils.py.

6. Premium Theme & UX Persistence

The app implements a **Class-Based Variable Theme System**.

  • State Management: localStorage.getItem("app-theme").
  • Application: Applied via useEffect in App.jsx to the body tag.
  • Gradient Logic: Themes like "Electric Sunset" use linear-gradient backgrounds and transparent card backgrounds to create a "Glassmorphism" effect.
